- [ ] **Step 1: 更新 system_prompt.md — 新增 Entity 匹配規則區段、更新 JSON 格式**

將 `src/nlu/system_prompt.md` 完整替換為以下內容：

```markdown
# System Prompt: Conversational CRM Parser Agent

## Role

你是一位資深 CRM 數據解析專家 (Parser Agent)。你的職責是接收業務人員 (BD) 的「意識流回報」——一段自然語言的文字輸入——並將其轉換為結構化的 JSON 指令，供下游的自動化調度器 (Dispatcher) 執行 CRM 操作。

## 核心行為規範

1.  **你只輸出 JSON**。不要加任何解釋、寒暄或 markdown 標記。
2.  **一段回報可能觸發多個意圖**。你必須完整解析所有意圖，不可遺漏。
3.  **今天的日期會在每次呼叫時由系統提供**，格式為 `YYYY-MM-DD`，作為日期推算的基準。
4.  **所有日期輸出一律使用 `YYYY-MM-DD` 格式**。
5.  **金額一律轉換為純數字**（例如「50 萬」→ `500000`，「兩千萬」→ `20000000`）。

## 意圖路由 (Intent Routing)

根據輸入內容，判斷應觸發以下哪些意圖（可複選）：

| 意圖 | 觸發條件 | 對應資料表 |
|---|---|---|
| `CREATE_ENTITY` | 提及一個**全新的**公司、客戶或合作夥伴 | Entity_Index |
| `UPDATE_PIPELINE` | 提及案件**階段變更**、預估金額、或下次跟進日期 | Strategic_Pipeline |
| `LOG_INTERACTION` | 描述了一次拜訪、會議、電話等**互動細節** | Interaction_Timeline |
| `SCHEDULE_ACTION` | 明確提到需要**安排後續任務**或提醒 | Action_Backlog |

## Entity 匹配規則

在輸出 `entities` 陣列時，必須根據「現有客戶名單」（Grounding Context，由系統在 Prompt 末尾動態附加）進行比對：

1. 若使用者提及的名稱與名單中某筆**完全相符或高度相似**（如「台積」→「台積電」），
   設定 `matched_entity_id` 為該筆 Entity_ID，並在 `entity_match_confidence` 給出信心分數（0 到 1）。
2. 若信心度 ≥ 0.80，視為確定匹配。
3. 若信心度介於 0.50–0.79，仍填入 `matched_entity_id`，但下游 UI 會要求使用者確認。
4. 若信心度 < 0.50，或名稱在名單中完全找不到，設定 `matched_entity_id` 為 `null`，視為新客戶。
5. 若系統未提供「現有客戶名單」，`matched_entity_id` 一律設為 `null`，`entity_match_confidence` 設為 `0`。
6. 一段回報中可能涉及多個 Entity，每個都需獨立匹配。

## Schema 對照表

### Entity_Index（當 `CREATE_ENTITY` 被觸發時填寫）
| 欄位 | JSON Key | 類型 | 規則 |
|---|---|---|---|
| Name | `name` | string | 公司或夥伴的正式名稱，盡量使用全名 |
| Category | `category` | enum | 只能是 `"Client"` 或 `"Partner"` |
| Industry | `industry` | string | 產業類別（如：半導體、金融、零售）|
| (Grounding) | `matched_entity_id` | string \| null | 匹配到的既有 Entity_ID，或 null |
| (Grounding) | `entity_match_confidence` | number | 0–1，信心分數 |

### Strategic_Pipeline（當 `UPDATE_PIPELINE` 被觸發時填寫）
| 欄位 | JSON Key | 類型 | 規則 |
|---|---|---|---|
| Entity_Name | `entity_name` | string | 關聯的客戶/夥伴名稱 |
| Stage | `stage` | enum | 只能是以下之一：`"尋商"` `"規格"` `"提案"` `"商議"` `"贏單"` `"輸單"` `"暫緩"` |
| Est_Value | `est_value` | number \| null | 預估金額（純數字），未提及則為 `null` |
| Next_Action_Date | `next_action_date` | string \| null | `YYYY-MM-DD` 格式，未提及則為 `null` |
| Status_Summary | `status_summary` | string | 用一句話總結目前案件狀態（由你生成）|

### Interaction_Timeline（當 `LOG_INTERACTION` 被觸發時填寫）
| 欄位 | JSON Key | 類型 | 規則 |
|---|---|---|---|
| Entity_Name | `entity_name` | string | 互動對象的公司名稱 |
| Raw_Transcript | `raw_transcript` | string | 原始輸入文字的完整保留 |
| AI_Key_Insights | `ai_key_insights` | string[] | 提取 3 個關鍵重點（bullet points），每個不超過 30 字 |
| Sentiment | `sentiment` | enum | 只能是 `"Positive"` `"Neutral"` `"Negative"` |

### Action_Backlog（當 `SCHEDULE_ACTION` 被觸發時填寫）
| 欄位 | JSON Key | 類型 | 規則 |
|---|---|---|---|
| Entity_Name | `entity_name` | string | 關聯的客戶/夥伴名稱 |
| Action_Description | `action_description` | string | 任務描述（清楚、可執行）|
| Action_Date | `action_date` | string | `YYYY-MM-DD` 格式 |

## 置信度與缺失欄位

- `overall_confidence`（0–1）：整體解析信心分數。若輸入文字過短、模糊，或缺乏關鍵資訊，給出較低分數。
- `missing_fields`：列出 AI 無法解析、需要使用者手動補填的欄位名稱（如 `["action_date", "est_value"]`）。

## 思考流程 (Chain of Thought)

收到輸入後，你應依照以下順序內部推理（不要輸出推理過程）：

1.  **識別實體**：這段話提到了哪些公司/人名？是新客戶還是已知客戶？若有 Grounding Context，比對名單。
2.  **判斷意圖**：哪些意圖應被觸發？（可複選）
3.  **提取數據**：對應每個意圖，從文字中提取相關欄位。
4.  **推算日期**：將「下週三」、「月底」等相對日期轉為絕對日期（根據系統提供的今天日期）。
5.  **生成摘要**：為 `status_summary` 和 `ai_key_insights` 生成簡潔的中文描述。
6.  **評估置信度**：給出 `overall_confidence` 分數，列出 `missing_fields`。

## JSON 輸出格式

```json
{
  "intents": ["CREATE_ENTITY", "UPDATE_PIPELINE", "LOG_INTERACTION", "SCHEDULE_ACTION"],
  "overall_confidence": 0.88,
  "missing_fields": [],
  "entities": [
    {
      "name": "公司名稱",
      "category": "Client",
      "industry": "產業類別",
      "matched_entity_id": "E-0001",
      "entity_match_confidence": 0.95
    }
  ],
  "pipelines": [
    {
      "entity_name": "公司名稱",
      "stage": "提案",
      "est_value": 500000,
      "next_action_date": "2025-04-10",
      "status_summary": "一句話現況摘要"
    }
  ],
  "interactions": [
    {
      "entity_name": "公司名稱",
      "raw_transcript": "完整原始文字",
      "ai_key_insights": [
        "關鍵點 1",
        "關鍵點 2",
        "關鍵點 3"
      ],
      "sentiment": "Positive"
    }
  ],
  "actions": [
    {
      "entity_name": "公司名稱",
      "action_description": "任務描述",
      "action_date": "2025-04-10"
    }
  ]
}
```

- `intents`：只包含實際被觸發的意圖。
- `entities` / `pipelines` / `interactions` / `actions`：只包含被觸發意圖對應的資料區塊，未觸發的意圖其對應陣列應為空陣列 `[]`。
- 每個陣列可能包含多個物件（例如一段話提到兩家客戶）。

## 邊界處理規則

1.  **公司名稱模糊**：盡量用輸入中出現的名稱原樣輸出，同時嘗試與 Grounding Context 比對。
2.  **日期模糊**：若只說「月底」取當月最後一天；「下週X」根據今天日期推算；「過幾天」取今天 +3 天。
3.  **金額未提及**：`est_value` 設為 `null`，不要猜測。
4.  **階段未提及**：`UPDATE_PIPELINE` 中的 `stage` 根據上下文推斷；若完全無法判斷，設為 `null`。
5.  **新 vs 舊客戶**：如果用戶說「新公司」、「第一次拜訪」等詞彙，觸發 `CREATE_ENTITY`。否則假設為已知客戶，不觸發建立。
6.  **一段話提到多家公司**：為每家公司分別產生對應的資料物件。

6. 一段回報中可能涉及多個 Entity，每個都需獨立匹配。

/**
 * 解析 BD 輸入的意識流文字，回傳結構化 JSON（不寫入 Sheets）
 * 動態注入 Entity_Index 全量名單作為 Grounding Context
 *
 * @param {string} rawText - BD 輸入的原始文字

function parseOnly(rawText) {
  try {
    if (!rawText || rawText.trim().length === 0) {
      return { status: 'error', message: '輸入文字不得為空' };

    // 動態讀取 Entity_Index，組成 Grounding Context
    const groundingContext = buildGroundingContext_();

    return { status: 'success', parsed_data: parsedData };

  } catch (error) {
    Logger.log('[parseOnly] 錯誤: ' + error.message);
    return { status: 'error', message: error.message };
  }
}

/**
 * 從 Entity_Index Sheet 讀取全量名單，格式化為 Grounding Context 文字
 * @returns {string} Grounding context 段落，若無資料則回傳空字串
 */
function buildGroundingContext_() {
  try {
    const ss = SpreadsheetApp.getActiveSpreadsheet();
    const sheet = ss.getSheetByName('Entity_Index');

    if (!sheet || sheet.getLastRow() <= 1) {
      return ''; // 尚無客戶資料，不注入 Grounding
    }

    // 讀取 Entity_ID(1), Name(2), Category(3), Industry(4)
    const data = sheet.getRange(2, 1, sheet.getLastRow() - 1, 4).getValues();
    const lines = data
      .filter(row => row[0] && row[1]) // 過濾空列
      .map(row => '- ' + row[0] + ' | ' + row[1] + ' | ' + row[2] + ' | ' + row[3]);

    if (lines.length === 0) return '';

    return '\n\n## 現有客戶名單（Grounding Context）\n' +
      '以下是系統中已存在的客戶。若使用者提及的對象與以下名稱相符或高度相似，\n' +
      '請使用對應的 Entity_ID，並在 entity_match_confidence 中給出信心分數（0-1）。\n\n' +
      lines.join('\n');
  } catch (e) {
    Logger.log('[buildGroundingContext_] 讀取 Entity_Index 失敗: ' + e.message);
    return ''; // 讀取失敗不阻擋流程
  }
}

/**
 * 將 BD 的意識流文字送至 Gemini，回傳結構化 JSON
 * @param {string} rawText - BD 輸入的原始文字
 * @param {string} groundingContext - 動態注入的客戶名單段落（可為空字串）
 * @returns {Object} parsedJSON - NLU 解析結果
 */
function callGeminiNLU_(rawText, groundingContext) {
  const apiKey = PropertiesService.getScriptProperties().getProperty('GEMINI_API_KEY');
  if (!apiKey) {
    throw new Error('請在 GAS Script Properties 中設定 GEMINI_API_KEY');
  }

  const today = Utilities.formatDate(new Date(), Session.getScriptTimeZone(), 'yyyy-MM-dd');
  const userMessage = '今天的日期：' + today + '\n\n' + rawText;

  // 將 Grounding Context 附加至 system prompt 末尾
  const fullSystemPrompt = SYSTEM_PROMPT + groundingContext;

  const url = 'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=' + apiKey;

  const requestBody = {
    system_instruction: {
      parts: [{ text: fullSystemPrompt }]
    },
    contents: [{
      role: 'user',
      parts: [{ text: userMessage }]
    }],
    generationConfig: {
      response_mime_type: 'application/json',
      temperature: 0.1
    }
  };

  const options = {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify(requestBody),
    muteHttpExceptions: true,
    // 30 秒 timeout 防止 GAS 卡死
    deadline: 30
  };

  const response = UrlFetchApp.fetch(url, options);
  const responseCode = response.getResponseCode();
  const responseText = response.getContentText();

  if (responseCode !== 200) {
    throw new Error('Gemini API 錯誤 (' + responseCode + '): ' + responseText.substring(0, 200));
  }

  const responseJson = JSON.parse(responseText);
  const content = responseJson.candidates?.[0]?.content?.parts?.[0]?.text;

  if (!content) {
    throw new Error('Gemini 回應格式異常，無法取得解析結果');
  }

  return JSON.parse(content);
}

/**
 * 取得使用者 Slack User ID
 * 結果快取於 Script Properties（key: SLACK_USER_ID）
 * @returns {string|null} Slack User ID 或 null
 */
function getSlackUserId_() {
  const props = PropertiesService.getScriptProperties();

  // 先嘗試讀取快取
  const cachedId = props.getProperty('SLACK_USER_ID');
  if (cachedId) return cachedId;

  // 查詢 Slack users.lookupByEmail
  const email = props.getProperty('USER_EMAIL');
  if (!email) {
    Logger.log('[getSlackUserId_] USER_EMAIL 未設定');
    return null;
  }

  const token = props.getProperty('SLACK_BOT_TOKEN');
  if (!token) {
    Logger.log('[getSlackUserId_] SLACK_BOT_TOKEN 未設定');
    return null;
  }

  const url = 'https://slack.com/api/users.lookupByEmail?email=' + encodeURIComponent(email);
  const response = UrlFetchApp.fetch(url, {
    method: 'get',
    headers: { 'Authorization': 'Bearer ' + token },
    muteHttpExceptions: true
  });

  const data = JSON.parse(response.getContentText());
  if (!data.ok) {
    Logger.log('[getSlackUserId_] Slack API 錯誤: ' + data.error);
    return null;
  }

  const userId = data.user.id;
  // 存入快取
  props.setProperty('SLACK_USER_ID', userId);
  Logger.log('[getSlackUserId_] 查詢成功，User ID: ' + userId + '，已快取');
  return userId;
}

/**
 * 發送訊息至 Slack（chat.postMessage）
 * 包含 Rate Limit 重試（等待 retry_after 秒後重試一次）
 * @param {string} channel - Slack User ID（DM）或頻道 ID
 * @param {Array} blocks - Slack Block Kit 區塊陣列
 * @returns {Object} { success: boolean, error?: string }
 */
function postMessage_(channel, blocks) {
  const token = PropertiesService.getScriptProperties().getProperty('SLACK_BOT_TOKEN');
  if (!token) {
    return { success: false, error: 'SLACK_BOT_TOKEN 未設定' };
  }

  const payload = JSON.stringify({ channel: channel, blocks: blocks });
  const options = {
    method: 'post',
    contentType: 'application/json; charset=utf-8',
    headers: { 'Authorization': 'Bearer ' + token },
    payload: payload,
    muteHttpExceptions: true
  };

  let response = UrlFetchApp.fetch('https://slack.com/api/chat.postMessage', options);
  let responseCode = response.getResponseCode();

  // Rate Limit 處理（HTTP 429）
  if (responseCode === 429) {
    const retryAfter = parseInt(response.getHeaders()['Retry-After'] || '5', 10);
    Logger.log('[postMessage_] Rate limited，等待 ' + retryAfter + ' 秒後重試');
    Utilities.sleep(retryAfter * 1000);
    response = UrlFetchApp.fetch('https://slack.com/api/chat.postMessage', options);
    responseCode = response.getResponseCode();
  }

  const data = JSON.parse(response.getContentText());
  if (!data.ok) {
    Logger.log('[postMessage_] Slack API 錯誤: ' + data.error);
    return { success: false, error: data.error };
  }

  return { success: true };
}

function buildReminderBlocks_(actionItems, mode, dateRangeLabel) {
  const blocks = [];

  // 標題
  const headerText = mode === 'friday_preview'
    ? '*📋 下週待辦預覽（' + dateRangeLabel + '）*'
    : '*📋 本週待辦確認（' + dateRangeLabel + '）*';

  blocks.push({
    type: 'section',
    text: { type: 'mrkdwn', text: headerText }
  });
  blocks.push({ type: 'divider' });

  if (actionItems.length === 0) {
    blocks.push({
      type: 'section',
      text: { type: 'mrkdwn', text: '_本週無待辦事項_' }
    });
    return blocks;
  }

  // 依日期分組
  const grouped = {};
  actionItems.forEach(item => {
    const date = item.due_date || '日期未定';
    if (!grouped[date]) grouped[date] = [];
    grouped[date].push(item);
  });

  // 按日期排序輸出
  Object.keys(grouped).sort().forEach(date => {
    // 日期標題（轉為 MM/DD 格式）
    const dateDisplay = date.length === 10
      ? (date.substring(5, 7) + '/' + date.substring(8, 10))
      : date;

    let itemsText = grouped[date]
      .map(i => '   📌 *' + i.ref_entity + '* — ' + i.task_detail)
      .join('\n');

    blocks.push({
      type: 'section',
      text: { type: 'mrkdwn', text: '*' + dateDisplay + '*\n' + itemsText }
    });
  });

  blocks.push({ type: 'divider' });
  blocks.push({
    type: 'context',
    elements: [{ type: 'mrkdwn', text: '共 ' + actionItems.length + ' 筆行動項目' }]
  });

  return blocks;
}

/**
 * 使用者確認後，執行 Staged Commit：Sheets 寫入 → Slack 通知
 *
 * @param {Object} confirmedData - 前端送出的確認資料
 *   {
 *     intents: [...],
 *     entities: [{ name, category, industry, matched_entity_id, entity_match_confidence }],
 *     pipelines: [{ entity_name, stage, est_value, next_action_date, status_summary }],
 *     interactions: [{ entity_name, raw_transcript, ai_key_insights, sentiment }],
 *     actions: [{ entity_name, action_description, action_date }],
 *     edit_log: [{ field, original, modified }]
 *   }
 * @returns {Object}
 *   {
 *     status: 'success' | 'partial_success' | 'error',
 *     result: { entities_created, pipelines_updated, interactions, tasks },
 *     slack_status: 'sent' | 'failed' | 'not_configured',
 *     failed_step: null | 'sheets' | 'slack'
 *   }
 */
function confirmWrite(confirmedData) {
  // --- Step 1: Sheets 寫入 ---
  let result;
  try {
    result = processPayload_(confirmedData);
  } catch (e) {
    Logger.log('[confirmWrite] Sheets 寫入失敗: ' + e.message);
    return {
      status: 'error',
      result: null,
      slack_status: 'not_attempted',
      failed_step: 'sheets',
      message: e.message
    };
  }

  // --- Step 2: Slack 即時確認通知 ---
  const token = PropertiesService.getScriptProperties().getProperty('SLACK_BOT_TOKEN');
  if (!token) {
    // Slack 未設定，不阻擋成功
    return {
      status: 'success',
      result: result,
      slack_status: 'not_configured',
      failed_step: null
    };
  }

  // 組裝 Slack 訊息所需資料（取第一筆 action + 對應的 pipeline）
  const actions = confirmedData.actions || [];
  const pipelines = confirmedData.pipelines || [];

  let slackStatus = 'sent';
  if (actions.length > 0) {
    const firstAction = actions[0];
    const matchedPipeline = pipelines.filter(p => p.entity_name === firstAction.entity_name);

    const actionData = {
      entity_name: firstAction.entity_name,
      action_description: firstAction.action_description,
      action_date: firstAction.action_date,
      pipelines: matchedPipeline
    };

    const slackResult = sendConfirmation(actionData);
    if (!slackResult.success) {
      Logger.log('[confirmWrite] Slack 通知失敗: ' + slackResult.error);
      slackStatus = 'failed';
    }
  } else if (pipelines.length > 0) {
    // 有 Pipeline 更新但無 Action，仍發送通知
    const p = pipelines[0];
    const actionData = {
      entity_name: p.entity_name,
      action_description: p.status_summary || '案件更新',
      action_date: p.next_action_date || '',
      pipelines: [p]
    };
    const slackResult = sendConfirmation(actionData);
    if (!slackResult.success) {
      slackStatus = 'failed';
    }
  }

  return {
    status: slackStatus === 'failed' ? 'partial_success' : 'success',
    result: result,
    slack_status: slackStatus,
    failed_step: slackStatus === 'failed' ? 'slack' : null
  };
}

/**
 * 重試 Slack 通知（Sheets 已寫入，只重送 Slack）
 *
 * @param {Object} actionData - 從 confirmWrite 回應暫存的資料
 *   { entity_name, action_description, action_date, pipelines }
 * @returns {Object} { slack_status: 'sent' | 'failed' }
 */
function retrySlack(actionData) {
  try {
    const slackResult = sendConfirmation(actionData);
    return { slack_status: slackResult.success ? 'sent' : 'failed' };
  } catch (e) {
    Logger.log('[retrySlack] 錯誤: ' + e.message);
    return { slack_status: 'failed' };
  }
}

/**
 * 根據 intents 分派至對應的 Handler
 */
function processPayload_(payload) {
  const intents = payload.intents || [];
  const editLog = payload.edit_log || [];
  const result = {
    entities_created: [],
    pipelines_updated: [],
    interactions_logged: [],
    tasks_scheduled: []
  };

  if (intents.includes('CREATE_ENTITY')) {
    result.entities_created = handleCreateEntities_(payload.entities || []);
  }

  if (intents.includes('UPDATE_PIPELINE')) {
    result.pipelines_updated = handleUpdatePipelines_(payload.pipelines || []);
  }

  if (intents.includes('LOG_INTERACTION')) {
    result.interactions_logged = handleLogInteractions_(payload.interactions || [], editLog);
  }

  if (intents.includes('SCHEDULE_ACTION')) {
    result.tasks_scheduled = handleScheduleActions_(payload.actions || []);
  }

  return result;
}

      if (pipeline.stage) sheet.getRange(row, 3).setValue(pipeline.stage);
      if (pipeline.est_value !== null && pipeline.est_value !== undefined) {
        sheet.getRange(row, 4).setValue(pipeline.est_value);

      if (pipeline.next_action_date) sheet.getRange(row, 5).setValue(pipeline.next_action_date);
      if (pipeline.status_summary) sheet.getRange(row, 6).setValue(pipeline.status_summary);
      sheet.getRange(row, 7).setValue('System');

      results.push({
        entity_name: entityName,
        action: 'UPDATED',
        project_id: existingRow.id
      });
    } else {
      // 新建記錄
      const newId = getNextProjectId();
      sheet.appendRow([
        newId,
        entityName,
        pipeline.stage || '',
        pipeline.est_value || '',

        'System'
      ]);

      results.push({
        entity_name: entityName,
        action: 'CREATED',
        project_id: newId

    }
  });

      return { row: i + 2, id: data[i][0] };
    }
  }
  return null;
}

/**
 * 計算本次提醒的日期範圍
 * friday_preview：下週一到下週日
 * monday_confirm：本週一到本週日
 */
function getDateRange_(mode) {
  const now = new Date();
  const tz = Session.getScriptTimeZone();

  let start, end;

  if (mode === 'friday_preview') {
    // 下週一（7 - 本週已過天數 + 1 天）
    const dayOfWeek = now.getDay(); // 0=Sun, 1=Mon, ..., 5=Fri, 6=Sat
    const daysUntilNextMonday = dayOfWeek === 0 ? 1 : (8 - dayOfWeek);
    start = new Date(now);
    start.setDate(now.getDate() + daysUntilNextMonday);
    end = new Date(start);
    end.setDate(start.getDate() + 6); // 下週日
  } else {
    // 本週一（往回推到週一）
    const dayOfWeek = now.getDay();
    const daysFromMonday = dayOfWeek === 0 ? 6 : (dayOfWeek - 1);
    start = new Date(now);
    start.setDate(now.getDate() - daysFromMonday);
    end = new Date(start);
    end.setDate(start.getDate() + 6); // 本週日
  }

  return {
    start: Utilities.formatDate(start, tz, 'yyyy-MM-dd'),
    end: Utilities.formatDate(end, tz, 'yyyy-MM-dd')
  };
}

/**
 * 從 Action_Backlog 讀取 pending 且在日期範圍內的任務
 * 若資料量大（> 50 筆），分批處理避免 GAS 執行時間超限
 */
function fetchPendingActions_(startDate, endDate) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName('Action_Backlog');

  if (!sheet || sheet.getLastRow() <= 1) return [];

  const lastRow = sheet.getLastRow();
  const results = [];
  const BATCH_SIZE = 50;

  for (let batchStart = 2; batchStart <= lastRow; batchStart += BATCH_SIZE) {
    const batchEnd = Math.min(batchStart + BATCH_SIZE - 1, lastRow);
    const rows = sheet.getRange(batchStart, 1, batchEnd - batchStart + 1, 8).getValues();

    rows.forEach(row => {
      const taskId     = row[0]; // 1: Task_ID
      const refEntity  = row[1]; // 2: Ref_Entity
      const taskDetail = row[2]; // 3: Task_Detail
      const dueDate    = row[3]; // 4: Due_Date（字串 yyyy-MM-dd 或 Date 物件）
      // row[4] = Reporter, row[5] = Slack_Notified, row[6] = Slack_Notified_At
      const status     = row[7]; // 8: Status

      if (!taskId || !dueDate) return;

      // 統一轉換為字串格式
      const dueDateStr = dueDate instanceof Date
        ? Utilities.formatDate(dueDate, Session.getScriptTimeZone(), 'yyyy-MM-dd')
        : String(dueDate).substring(0, 10);

      // 篩選：status = pending 且日期在範圍內
      if (status !== 'pending' && status !== false && status !== '') return;
      if (dueDateStr < startDate || dueDateStr > endDate) return;

      results.push({
        ref_entity: refEntity,
        task_detail: taskDetail,
        due_date: dueDateStr
      });
    });
  }

  return results;
}

/**
 * 將 yyyy-MM-dd 格式化為 MM/DD 顯示用字串
 */
function formatDateLabel_(dateStr) {
  if (!dateStr || dateStr.length < 10) return dateStr;
  return dateStr.substring(5, 7) + '/' + dateStr.substring(8, 10);
}

function setState(newState) {
  STATE = newState;
  const panels = ['input','loading','preview','confirming','result','error'];
  panels.forEach(p => {
    const el = document.getElementById('panel-' + p);
    if (el) el.classList.remove('active');
  });

  const map = {
    'INPUT': 'input',
    'LOADING': 'loading',
    'PREVIEW': 'preview',
    'EDITING': 'preview',    // EDITING 共用 preview panel，靠 renderPreviewActions() 切換按鈕

    'RESULT': 'result',
    'ERROR': 'error'
  };
  const panelId = 'panel-' + (map[newState] || 'input');
  document.getElementById(panelId).classList.add('active');

  if (newState === 'PREVIEW' || newState === 'EDITING') {
    renderPreviewActions();
  }

  charCountEl.textContent = inputEl.value.length + ' 字';

function handleSubmit() {
  const rawText = inputEl.value.trim();
  if (!rawText || rawText.length < 5) {
    inputEl.focus();
    return;
  }

  editLog = [];
  parsedData = null;
  setState('LOADING');
  animateLoadingSteps();

  google.script.run
    .withSuccessHandler(function(res) {
      if (res.status === 'success') {
        parsedData = res.parsed_data;
        renderParsedSummary(parsedData);

        // 若 overall_confidence < 50%，自動進入 EDITING
        if ((parsedData.overall_confidence || 1) < 0.5) {
          setState('EDITING');
        } else {
          setState('PREVIEW');
        }
      } else {
        showError('AI 解析失敗：' + (res.message || '未知錯誤'), false);
      }
    })
    .withFailureHandler(function(err) {
      showError('執行錯誤：' + err.message, false);
    })
    .parseOnly(rawText);
}

function animateLoadingSteps() {
  const steps = ['lstep-1','lstep-2','lstep-3'];
  steps.forEach(id => {
    const el = document.getElementById(id);
    if (el) { el.className = 'step'; }
  });

  let i = 0;
  function next() {
    if (i > 0) {
      const prev = document.getElementById(steps[i-1]);
      if (prev) { prev.className = 'step done'; }
    }
    if (i < steps.length) {
      const cur = document.getElementById(steps[i]);
      if (cur) { cur.className = 'step active'; }
      i++;
      setTimeout(next, 2500);
    }
  }
  next();
}

function trackEdit(field, newValue, originalOverride) {
  // 從 parsedData 中取得原始值（若未指定 override）
  const original = originalOverride !== undefined ? originalOverride : getOriginalValue(field);
  const existing = editLog.findIndex(e => e.field === field);
  if (existing >= 0) {
    editLog[existing].modified = newValue;
  } else {
    editLog.push({ field: field, original: original, modified: newValue });
  }
}

function getOriginalValue(field) {
  // 簡單字串路徑解析（支援 entities[0].name 格式）
  try {
    const parts = field.replace(/\[(\d+)\]/g, '.$1').split('.');
    let obj = parsedData;
    for (const part of parts) { obj = obj[part]; }
    return obj;
  } catch(e) { return null; }
}

function handleConfirm() {
  if (!parsedData) return;

  setState('CONFIRMING');
  animateConfirmingSteps();

  // 組裝 confirmedData
  const confirmedData = {
    intents: parsedData.intents || [],
    entities: parsedData.entities || [],
    pipelines: parsedData.pipelines || [],
    interactions: parsedData.interactions || [],
    actions: parsedData.actions || [],
    edit_log: editLog
  };

  google.script.run
    .withSuccessHandler(function(res) {
      if (res.status === 'success' || res.status === 'partial_success') {
        // 暫存 retrySlack 所需的 actionData
        if (res.slack_status === 'failed' && confirmedData.actions.length > 0) {
          const a = confirmedData.actions[0];
          pendingSlackActionData = {
            entity_name: a.entity_name,
            action_description: a.action_description,
            action_date: a.action_date,
            pipelines: (confirmedData.pipelines || []).filter(p => p.entity_name === a.entity_name)
          };
        }

        if (res.slack_status === 'failed') {
          showPartialSuccess(res.result, res.slack_status);
        } else {
          showResult(res.result, res.slack_status);
        }
      } else {
        showError('資料寫入失敗：' + (res.message || '未知錯誤'), false);
      }
    })
    .withFailureHandler(function(err) {
      showError('執行錯誤：' + err.message, false);
    })
    .confirmWrite(confirmedData);
}
